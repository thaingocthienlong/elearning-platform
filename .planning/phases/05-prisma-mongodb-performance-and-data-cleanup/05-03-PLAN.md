# 05-03-PLAN: Ticket Diagnostics And Watermark Singleton Behavior

## Objective

Document support diagnostic bounds and move watermark settings toward explicit singleton behavior without destructive cleanup.

## Requirements

- `DATA-05`: Ticket diagnostics and security events have documented retention or payload bounds.
- `DATA-06`: Watermark settings update/read behavior avoids ambiguous latest-row selection or unbounded history growth.

## Tasks

1. Document support ticket diagnostic limits from `src/lib/request-security.ts`.
2. Add explicit singleton key behavior for `WatermarkSettings` using a stable `scope` field and unique index.
3. Update admin/public/Zoom watermark readers to use the singleton scope.
4. Update admin watermark writes to upsert the singleton document rather than append a new row.
5. Add focused tests for watermark singleton create/update/read behavior.

- `npm run prisma:generate`
