# 05-02-PLAN: Bounded Admin Analytics And Security Events

## Objective

Bound admin analytics and security event reads while preserving response shapes.

## Requirements

- `DATA-03`: Admin analytics queries use bounded ranges, pagination, aggregation, or short-lived caching.
- `DATA-04`: Indexes support admin and security-event query shapes.
- `DATA-05`: Security events have documented retention or payload bounds.

## Tasks

1. Add date/limit bounds to admin analytics reads and avoid broad all-user/all-record reductions where possible.
2. Add a short-lived cache for the admin analytics response if low risk.
3. Clamp security-event `page` and `limit`, add optional date-window bounds, and preserve existing pagination response shape.
4. Document admin analytics and security-event bounds.
5. Add focused route/helper tests for limit/date bound behavior where practical.
