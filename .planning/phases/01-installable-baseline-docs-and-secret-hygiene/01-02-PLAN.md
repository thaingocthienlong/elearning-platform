## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local shell -> verification scripts | Maintainer executes scripts that inspect local files and env names. |
| env files -> service verifier | Service verifier loads env variable names and presence, but must not print values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-03 | Tampering | `scripts/verify-services.ts` | mitigate | Implement `--strict` and `CI=true` fail behavior so staging gaps cannot be hidden by local optional checks. |
| T-1-04 | Information Disclosure | `scripts/verify-services.ts` | mitigate | Print missing variable names only; never print env values, masks, tokens, or connection strings. |
| T-1-06 | Repudiation | `docs/verification.md` | mitigate | Document exact commands and expected skip/fail semantics so verification results can be reproduced. |
</threat_model>
