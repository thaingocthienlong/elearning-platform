<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| package scripts -> local shell | Maintainer runs root scripts that execute repository TypeScript and external CLIs. |
| Jest setup -> test process env | Mock values are injected into process env for tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | `jest.setup.ts` | mitigate | Preserve mock-only comments and only add test-library import; do not copy real env values. |
| T-1-03 | Denial of Service | `package.json` verification scripts | mitigate | Expose default and strict service verification scripts separately so local setup can skip credentials while strict mode can fail. |
| T-1-05 | Tampering | `package-lock.json` | mitigate | Use npm install commands to update the lockfile rather than hand-editing dependency resolution. |
</threat_model>
