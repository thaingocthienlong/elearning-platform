# 06-03 Plan: Staging Smoke Checklist and Verifier

## Goal

Create a staging smoke checklist and a non-secret verifier that proves the checklist covers the required external and user-facing flows.

## Requirements

- STAGE-05
- TEST-07

## Scope

- Add `docs/staging-smoke-checklist.md`.
- Add `scripts/verify-staging-smoke.ts`.
- Add package script `verify:staging`.
- Checklist statuses must allow:
  - `not run`
  - `pass`
  - `fail`
  - `blocked: missing credentials/service access`
- Smoke IDs must cover auth, course access, playback, DRM token issuance, HLS access, Zoom meeting launch, support ticket creation, Redis, storage, Axinom webhook readiness, logs, Sentry, and production-gap review.

## Verification

- Run `npm run verify:staging`.
- Add Jest coverage for the docs/script contract.

## Status

Planned.
