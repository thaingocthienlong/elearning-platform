# 04-01-PLAN: Server-Owned Zoom Signature Contract

## Objective

Make `/api/zoom/signature` enforce a server-owned Zoom Meeting SDK signature contract for `ZOOM-03`, `ZOOM-04`, and `TEST-04`.

## Requirements

- `ZOOM-03`: Zoom signatures are generated server-side only and never expose SDK secrets.
- `ZOOM-04`: Learners cannot mint host-capable signatures.
- `TEST-04`: Signature behavior is covered by automated tests.

## Tasks

1. Validate the NextAuth session before reading Zoom signing inputs or generating any JWT.
2. Stop trusting client-supplied `meetingNumber` and `role`; use the configured meeting ID and derive role from `session.user.role`.
3. Fail closed when `ZOOM_MEETING_SDK_KEY`, `ZOOM_MEETING_SDK_SECRET`, or `NEXT_PUBLIC_ZOOM_MEETING_ID` is missing.
4. Return only browser-safe fields: signature, SDK key, server-owned meeting number, public passcode, derived role, and watermark settings.
5. Add node-environment API tests for unauthenticated denial, missing config denial, learner role `0`, admin role `1`, and no SDK secret leakage.
