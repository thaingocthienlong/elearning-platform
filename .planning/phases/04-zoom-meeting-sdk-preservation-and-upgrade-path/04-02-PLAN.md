# 04-02-PLAN: Meeting Page Minimal Payload And Error States

## Objective

Preserve the authenticated `/meeting` iframe flow while making the browser request minimal and user-safe for `ZOOM-01`, `ZOOM-03`, and `ZOOM-06`.

## Requirements

- `ZOOM-01`: Current authenticated meeting flow is documented.
- `ZOOM-03`: Browser does not control signing-sensitive values.
- `ZOOM-06`: Meeting launch smoke path preserves the current user flow.

## Tasks

1. Change `src/app/meeting/page.tsx` so it sends a minimal signature request body.
2. Use server-returned meeting number/passcode/role data when building the iframe URL.
3. Preserve identity and watermark URL parameters used by `public/zoom-meeting.html`.
4. Improve unavailable/error text without leaking operational secret/config details.
