# Plan 03-03: Shaka Axinom License Request Integration

**Wave:** 1
**Depends on:** 03-01
**Requirements:** DRM-05

## Goal

Shaka playback uses documented Axinom license URLs and sends entitlement tokens only with DRM license requests.

## Tasks

1. Add `src/lib/shaka-axinom.ts` with pure helpers for:
   - Widevine, PlayReady, and FairPlay default license URLs.
   - env override resolution for public license URLs.
   - determining whether a Shaka request should receive `X-AxDRM-Message`.
2. Update `src/hooks/player/useShakaPlayer.ts` to consume the pure helper and reduce noisy DRM console logs in touched code.
3. Update `src/components/video/DRMPlayerWrapper.tsx` if needed so PlayReady has a documented/default path consistent with env docs.
4. Preserve existing FairPlay certificate fetching, but document that real FPS credentials/certificates come from Apple/Axinom setup and are not invented locally.
5. Add focused tests for helper behavior.

## Verification

- `npm test -- shaka-axinom --runInBand`
- `npm run typecheck`

## Done When

- `X-AxDRM-Message` is test-proven to apply only to license requests.
- Public license URL defaults and overrides are documented and tested.
- Client logs no longer expose token or tenant operational details in touched paths.
