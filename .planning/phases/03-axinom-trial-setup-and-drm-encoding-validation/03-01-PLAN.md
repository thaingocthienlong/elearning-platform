# Plan 03-01: Axinom Official Setup Docs and Env Validation

**Wave:** 0
**Requirements:** DRM-01, DRM-02, DRM-03

## Goal

Maintainers can follow official Axinom DRM/Encoding setup documentation and map portal values to canonical repo env vars without relying on inherited env files.

## Tasks

1. Create `docs/axinom-setup.md` with official-doc links and step-by-step sections for:
   - DRM trial/portal setup.
   - Communication Key ID/value handling.
   - License service URLs for Widevine, PlayReady, and FairPlay.
   - Entitlement Message and License Service Message concepts.
   - Encoding UI-first setup: storage, acquisition/publishing profiles, processing profiles, upload, encode, preview.
   - Credentials protection for storage/DRM secrets.
   - Webhook URL and staging callback notes.
2. Add or update Axinom env rows in `.env.example` and `docs/env-matrix.md`, including `NEXT_PUBLIC_AX_PR_LS_URL` and canonical-vs-legacy notes.
3. Add `src/lib/axinom-env.ts` for canonical Axinom env validation:
   - local mode returns warnings.
   - strict mode fails missing canonical values.
   - legacy `AX_*` aliases are reported.
   - public license URL defaults are available.
4. Update `scripts/verify-axinom-setup.ts` to consume the new env validator without printing secret values.
5. Add Jest tests for `axinom-env`.

## Verification

- `npm test -- axinom-env --runInBand`
- `npm run typecheck`

## Done When

- Axinom docs cite official sources and contain no real secret values.
- Canonical env validation is tested.
- Strict validation catches missing/ambiguous Axinom setup.
