# Plan 03-04: Explicit Axinom Video Metadata Fields

**Wave:** 2
**Depends on:** 03-01
**Requirements:** DRM-08

## Goal

Video records store Axinom operational state in explicit fields instead of overloading user-facing descriptions.

## Tasks

1. Update `prisma/schema.prisma` `Video` model with explicit optional fields:
   - DRM Axinom video ID.
   - clear Axinom video ID or retain and normalize existing `axinomIdClear`.
   - Axinom encoding status.
   - Axinom job/output identifiers where available.
   - last sync timestamp.
2. Run Prisma generation after schema update.
3. Update `src/app/api/video/process/route.ts` to write explicit Axinom fields and preserve existing description text.
4. Update `src/lib/axinom-sync.ts` to prefer explicit fields and keep legacy description parsing only as fallback for inherited records.
5. Update `src/app/api/webhook/axinom/route.ts` to find videos by explicit Axinom IDs first and update explicit status/sync fields.
6. Add focused tests for explicit-field preference and legacy fallback where practical.

## Verification

- `npm run prisma:generate`
- `npm test -- axinom-sync --runInBand`
- `npm test -- webhook-and-admin-security --runInBand`
- `npm run typecheck`

## Done When

- New records no longer require `description` parsing for Axinom operational flow.
- Existing legacy records can still sync through fallback behavior.
- Prisma client generation succeeds.
