# Plan 03-05: Axinom Staging Runbook and Phase Gate

**Wave:** 3
**Depends on:** 03-02, 03-03, 03-04
**Requirements:** DRM-01, DRM-02, DRM-03, DRM-04, DRM-05, DRM-06, DRM-07, DRM-08, DRM-09

## Goal

Maintainers have a tenant-safe staging procedure to encode, publish, authorize, and play a real Axinom-protected test video without forcing live credentials into local automation.

## Tasks

1. Create `docs/axinom-staging-checklist.md` covering:
   - required tenant/service-account values.
   - strict env verification.
   - test video upload/encoding.
   - webhook callback configuration.
   - DB field checks.
   - authorized playback and browser network checks.
   - expected failure diagnostics without secret exposure.
2. Update `scripts/verify-services.ts` or `scripts/verify-axinom-setup.ts` docs/output so Axinom live checks remain opt-in and strict-mode friendly.
3. Add/update root docs links so maintainers can find Axinom setup and staging checks from setup documentation.
4. Run focused tests and full phase gate.
5. Write `03-VERIFICATION.md`, update requirements, roadmap, and state when implementation passes.

## Verification

- `npm run lint`
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run secrets:scan`

## Done When

- Automated gates pass.
- Axinom staging validation is documented as manual/opt-in.
- Phase 3 requirements can be marked complete without pretending a live tenant was exercised locally.
