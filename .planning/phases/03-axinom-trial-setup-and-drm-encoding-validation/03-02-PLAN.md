# Plan 03-02: Official License Service Message Signing and Local License Quarantine

**Wave:** 1
**Depends on:** 03-01
**Requirements:** DRM-04, DRM-06, DRM-07

## Goal

Authorized playback receives server-generated Axinom License Service Message JWTs shaped according to official docs, while the local license endpoint is clearly non-production.

## Tasks

1. Refactor `src/lib/axinom.ts`:
   - Build a testable Entitlement Message and License Service Message payload.
   - Include authorized key IDs only.
   - Add explicit short expiration inside the license message.
   - Use base64-decoded communication key and HS256 signing.
   - Remove unsafe token/key console logging.
2. Update `src/app/api/drm/token/route.ts` to call the centralized helper and return sanitized failures.
3. Quarantine `src/app/api/drm/license/route.ts`:
   - Keep entitlement check if request includes video ID.
   - Return `501 Not Implemented` or equivalent explicit non-production response after authorization.
   - Link behavior to Axinom standard mode in comments/docs without exposing internals.
4. Add unit and route tests:
   - payload shape and validation.
   - unauthorized route does not sign.
   - authorized route signs with expected key IDs.
   - local license route does not pretend to issue keys.

## Verification

- `npm test -- axinom --runInBand`
- `npm test -- media-routes --runInBand`
- `npm run typecheck`

## Done When

- License Service Message generation is centralized and official-doc-shaped.
- The browser never receives communication keys.
- The local DRM license route cannot be mistaken for production Axinom DRM.
