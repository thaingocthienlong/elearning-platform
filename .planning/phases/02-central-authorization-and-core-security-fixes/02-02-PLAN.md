<objective>
Adopt the shared entitlement helper across watch, DRM, HLS, local license, and heartbeat surfaces so protected media behavior is consistent.
</objective>

<must_haves>
- HLS playlist requests are denied unless the same entitlement rules as the watch page pass.
- DRM token issuance respects direct-access validity windows and view limits.
- Local DRM license route requires entitlement when a `videoId` is supplied and remains clearly non-production/stubbed for key issuance.
- Heartbeat updates cannot create or increment watch records for users who are no longer entitled.
- Denial responses use the sanitized HTTP mapping from Plan 02-01.
- Existing success paths for authorized users continue to work.
</must_haves>
