<objective>
Harden malformed webhook handling, security-event destructive action protection, and redacted logging on touched security routes.
</objective>

<must_haves>
- Malformed or wrong-length Axinom webhook signatures return controlled 403 responses, not 500.
- Security-event flush requires explicit confirmation and records admin identity audit metadata.
- Touched routes use a redacted server logger instead of raw `console.*` for sensitive operational failures.
- Secret hygiene from Phase 1 is preserved; no new logging or docs expose raw secrets.
- Tests cover malformed webhook signatures and security-event flush confirmation/audit behavior.
</must_haves>

## Task 1: RED - Add webhook/admin security tests

**Type:** tdd
**Files:**
- `__tests__/api/webhook-and-admin-security.test.ts`

**Action:**
Add failing tests for:
- missing Axinom signature returns 401.
- malformed non-hex or wrong-length Axinom signature returns 403, not 500.
- invalid but well-shaped Axinom signature returns 403.
- security-event DELETE without admin returns 401.
- security-event DELETE without exact confirmation returns 400 and does not delete.
- security-event DELETE with confirmation writes an audit event or audit metadata before/after deletion.

Mock Prisma, NextAuth, and fetch for Axinom details.

**Verify:**
- `npm test -- webhook-and-admin-security --runInBand` fails against current behavior.

**Acceptance Criteria:**
- Tests prove current malformed signature and destructive flush gaps.
